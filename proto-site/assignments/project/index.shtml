Major Project

Project Overview

Using production methods discussed today and development techniques learned throughout the course, your group will create a proto-site and documents to support the redesign of the COMP1950 web site.

Requirements

The COMP 1950 owner (Alistair Calder) would like the web site redesigned in order to better meet the needs of students. The website owner has contracted your group, as students of the class, to act as contractors for this project.

Project Goals

  • Students must have easy access to notes, examples and other information
  • The site should reflect the aims of the course:
    • Standards compliant web development
    • CSS
    • HTML 5
    • Methodology-driven design, etc

Deliverables

  • For this project, you will create a web site that will act as the project home for all of the other documents you will create.
  • This is frequently referred to as a Client Site or a Cliet Project Site and is a place for clients to find information on team members, progress reports, artifacts, examples and more
  • Your client site will be similar to your course homepage, in that it is a central location for all the other documents you are making available
  • Your other deliverables are going to be:
    1. A Client Project Site created for this project that describes the project and links to various artifacts.
    2. An Introduction page that describes the project (as you understand it, including the goals of the project) and the members of the team. This page should also list what each team member actually did during the project
    3. A proto-site that has at least 3 different pages mocked up: the home page, a lecture page and the blog page
    4. A blog page must be included, and it must be on Tumblr. The template you use for the blog must look identical in layout and presentation to your protosite
    5. Wireframe(s) of the major sections of the site (Example lecture page) that you used to design the protosite (done in HTML, Word or Visio). Your designs may have started on paper but they should be ported to digital. These can also be done by using paper prototypes (scanned) or just using a low-tech solution like Word (make boxes in Drawing mode) or HTML (using simple tables)
    6. In the wireframes, include descriptions of what is included in each content area
    7. Sitemap (Example sitemap) (HTML, Word or Visio). the sitemap is for your proposed site, NOT the client site.
    8. Rationale page - this is the most important page in your site. This is where you describe why you chose to build the site in the way you have proposed. Make sure you agreee on this in your group!
    9. All of this should be presented in an HTML 5 compliant web site with CSS-based navigation

Answers

Answers to student questionnaires.

Teamwork

  • This project will require work from all members of your team
  • Start by agreeing on a plan of attack - get agreement!
  • Start making decisions on how the site will look, what functions it needs to have, how users will access information
  • Draw some sketches for how it could look, think about labels and information design
  • Once you know what needs to be on the site and how it will look, divide up the work. Make sure your team knows who is doing what
  • There is plenty to do, but if you work together, it is an easy project to get done in time
  • Think outside the box - don't just copy the current design. Look around at other sites and use your creativity. But - be realistic! Make your suggestions something that can work
  • If you have questions that will further refine your own direction, ask the owner of the site for details
  • Think about this project in context of the Methodology discussion that happened in class and the readings for week 8
  • Best suggestion: start from scratch with the content as a known variable. Build from there.

Validation

  • All content must be valid HTML 5
  • All presentation must be valid CSS
  • Add a link on each page to validate:
    • HTML: http://validator.w3.org/

Remember!

  • This project can be done with approximately 20-30 hours of total work. With 4 people on a team, and 8 hours of (total) work each, that equals 32 hours. Lots of time.
  • Look at the existing site, and break it down. What are the component parts? Begin there. What needs to be done? What should be there?
  • As a student, YOU are the audience. Talk to your cohorts, spend time considering the current experience, design the site the way you think it would work best.
  • You do not have to include actual notes for the class. This project is to produce a proof-of-concept of the new COMP 1950 site
  • Think outside the box! Be creative!
